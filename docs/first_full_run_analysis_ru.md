## 3) Сравнительный анализ: что изменилось между прогоном №1 и №2

1. **Стабильное лидерство PSO подтвердилось**: улучшение на control выросло с `+25.17%` до `+31.39%`.
2. **DE значительно улучшился**: `+16.11% -> +27.49%`, при этом оптимум сместился из «дальней» зоны (`B2/B1≈100`) в компактную (`B2/B1≈1.8`).
3. **GA стал одним из лидеров**: `+6.41% -> +29.29%`, оптимум близок к `B2≈B1`.
4. **RS перестал деградировать**: `-21.67% -> +20.94%`; значит, сужение окна поиска и больший бюджет действительно помогли убрать много неэффективных конфигураций.
5. **BO вышел почти в ноль**: `-59.88% -> -1.12%`, но пока уступает остальным методам и требует либо отдельной тонкой настройки, либо понижения приоритета.
6. По всем успешным методам полезная зона во втором прогоне стала ещё компактнее: `B1` в диапазоне ~`1000..1700`, а `B2/B1` чаще `1..10` (кроме BO, где граница почти упёрлась в `ratio_max=60`).

## 4) Выводы по двум прогонам

- Корректировки после первого прогона были **правильными**: качество выросло почти у всех методов, а worst-case деградации исчезли (кроме лёгкой просадки BO).
- Для следующего цикла имеет смысл сделать акцент на **PSO + GA + DE** как на основных методах.
- Пространство поиска можно продолжать сужать, так как оба прогона указывают на низкие значения `B1` и умеренное соотношение `B2/B1`.
