## Наблюдения

1. **Победитель прогона — PSO**, второй по стабильности — **DE**.
2. **RS и BO ухудшили контроль**, что указывает на шум/недоисследование при текущем бюджете и параметрах.
3. У DE/RS видно много «дорогих» оценок с очень крупными границами, что раздувает runtime без практической пользы.
4. По лучшим точкам разных методов полезная зона выглядит узкой: `B1` около `1e3..5e3`, а `B2/B1` чаще умеренное.

## Выводы для следующего плана

- Сузить поисковое окно (`B1/B2` + `ratio_max`), чтобы убрать заведомо дорогие и малополезные области.
- Увеличить бюджет у методов, которые недоисследуют пространство (DE/BO/GA/RS), но уже в **узком окне**.
- Для валидации повысить `curves-per-n`, чтобы уменьшить дисперсию метрик при сравнении с baseline.
- Сделать акцент на PSO и DE как на основных кандидатах для следующего углубленного прогона.
